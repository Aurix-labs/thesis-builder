以下是“交易复盘双子系统”的完整设计，力求可直接落地为两个 Skill 的交互式 Agent。设计遵循三大原则：**策略而非预测、过程可复盘、长期可迭代**。

---

## 一、总体架构与能力边界

两个 Skill 是一个决策链条的上下游：

- **市场复盘 Skill**（宏观观察员）：回答“明天适不适合交易？主战场在哪？该用几成力？”
- **个股复盘 Skill**（微观执行教练）：回答“在这只股票上，明天具体怎么打？打错了怎么撤？今天这笔交易是赚是赔在对什么？”

**边界铁律：市场复盘绝不提个股，个股复盘必须引用市场复盘的阶段结论作为前置参数。**

同时，系统内置一个**验证与迭代模块**，持续评估两个 Skill 的决策质量并输出优化建议。

---

## 二、市场复盘 Skill 设计

### 2.1 角色定义
你是专业的A股盘面分析员，负责输出当日市场全貌与次日作战环境评估。你只描述事实与推演场景，**绝不给出任何具体股票的买卖意见**。

### 2.2 输入数据清单
- 主要指数（上证、沪深300、中证500/1000、创业板指）的日K线、分时图、成交量
- 全市场上涨/下跌家数，涨幅>5%与跌幅<-5%家数
- 涨停板、跌停板、炸板率，连板高度与梯队序列
- 概念板块资金净流入、板块内涨停家数、核心股
- 北向资金当日净买卖额及行业分布
- 盘后重要政策、行业新闻、公告摘要（整理成影响评级）
- 昨日市场复盘Skill输出的场景推演（用于验证）

### 2.3 核心功能模块

**模块一：大盘环境诊断**
- 各指数多空状态（多头排列/空头排列/震荡，5、20日均线方向）
- 成交额所处量能台阶及与前日的对比
- 涨跌家数比，市场情绪温度计（极度恐惧/恐惧/中性/乐观/狂热）

**模块二：情绪周期定位**
- 确定当前周期阶段：冰点、修复、主升、高位分歧、退潮
- 辅证数据：最高连板高度及封单力度，昨日涨停今日溢价率，日内大面股数量

**模块三：主线与支线识别**
- 涨停股按概念归类，确定主线（涨停家数最多+梯队最完整+有容量中军）
- 判定主线状态：加强、分歧、转弱
- 支线定位（卡位轮动/新方向萌芽）及持续性评估

**模块四：资金行为监测**
- 北向资金行为：连续三日流向及主要加仓方向
- 龙虎榜关键信号：顶级游资与机构是否同向共振，或进入防守避险

**模块五：盘后变量汇总**
- 影响明日的预期差清单：重大政策、产业事件、海外市场异动、大宗商品等
- 对每个变量给出“影响级别”：决定全局/影响板块/仅影响个股

**模块六：明日作战地图（场景推演）**
- 基于上述所有结论，输出三种指数的明日推演路径：
  - 强势路径（触发条件、应对策略、适合总仓位区间）
  - 中性路径（触发条件、应对策略、仓位）
  - 弱势路径（触发条件、应对策略、仓位）
- 给出统一仓位建议：进攻（7成以上）、均衡（4-6成）、防御（2-3成）、空仓
- 风险提示：当前阶段最需要警惕的陷阱（如流动性衰竭、高标A杀、消息兑现）

### 2.4 输出格式要求
全部结论以结构化文本呈现，分为“今日复盘摘要”和“明日作战地图”两部分。所有判断必须附带客观数据依据。

## 三、个股复盘 Skill 设计

### 3.1 角色定义
你是私人交易教练，绑定用户的持仓与关注标的，结合市场环境，输出该股的深度复盘与次日精确作战计划。你同时负责对用户当日的实际交易行为进行纪律性评估。

### 3.2 输入数据清单
- 用户指定的个股代码、自身持仓成本、止损/止盈价（可选）
- 该股当日日K线、分时图、成交量、盘口数据
- 该股所属板块当日的强弱表现及内排名
- 该股盘后公告、新闻、大宗交易等
- **强制引用参数**：来自市场复盘 Skill 的当日情绪周期阶段、仓位建议、主线方向
- 用户当日该股的交割单（或手动输入“我在XX价位买了/卖了XX”）

### 3.3 核心功能模块

**模块一：技术与位置复盘**
- 在日K线图上标记最新的关键支撑（如涨停价、均线、筹码峰）和阻力（前高、缺口）
- 分析当日的分时攻击/防御质量：拉升时放量与否，回调是否缩量，尾盘是否异动
- 评判当日量能状态：属于攻击性放量、分歧放量还是衰竭缩量

**模块二：逻辑与地位校验**
- 识别该股核心驱动逻辑（由用户自定义或系统抓取），并结合盘后信息判断：强化、持平、或证伪
- 评定该股在所属板块内的当日强度：引领者、跟风者、掉队者
- 若为主线板块个股，对标板块内龙头表现，判断是“真强”还是“被动带起”

**模块三：交易纪律评估（核心差异模块）**
- 读取用户交割单，对比昨日该股预案，判定本次买卖的“计划执行度”
- 将交易归因为四类：
  - 模式内盈利 / 模式内亏损（正确的交易，无论盈亏）
  - 情绪化盈利 / 情绪化亏损（错误的交易，需反省）
- 输出一句直面心态的提问（如：“你当时买入，是因为信号，还是因为害怕错过？”）

**模块四：明日个股作战预案**
- 结合市场复盘Skill的环境参数，生成精准的导航式指令，必须包含：
  - **开仓/加仓条件**：精确到价格、时间、量能、盘口特征（例如：竞价高开不超3%，且9:35后回踩分时均线不破，可1成仓介入）
  - **持仓/减仓条件**：例如：冲高遇前高阻力区域量能跟不上，则减半仓
  - **止损/纠错条件**：例如：跌破今日最低价，无条件清仓
- 预案会自动适配市场环境：退潮期默认收紧触发条件、降低仓位；主升期适当放宽

### 3.4 输出格式要求
为每只个股输出独立卡片，包含：关键价位图示（文字描述）、逻辑状态、纪律归因、明日作战指令。作战指令用“如果……，则……”的编程化语言，不可模糊。

## 四、双 Skill 协同工作流

1. **盘后（15:30）**：先运行市场复盘 Skill，生成《作战地图》。
2. **随后**：运行个股复盘 Skill，系统自动将《作战地图》的核心结论（情绪阶段、仓位建议、主线板块）作为强制约束参数注入每个个股预案。
3. **产出**：个股 Skill 输出每只股票的《明日作战卡》。
4. **盘中**：用户可根据作战卡执行。期间不做新决策，除非出现《作战地图》中“超预期场景”的触发信号。

## 五、内置验证与迭代系统

为使 Agent 能力不断进化，系统中内嵌评估层，对两个 Skill 进行长期跟踪。

### 5.1 市场复盘 Skill 评估指标
每日盘后自动记录以下数据，并与昨日结论比对：
- 情绪周期判断准确率（与实际盘面涨幅5%、跌幅-5%家数趋势是否一致）
- 主线板块判断的延续性（今日主线是否仍在昨日主线名单内，或明确提示切换）
- 风险预警有效性（提示的退潮陷阱、高标风险是否真的发生）
- 场景推演覆盖度（实际走势是否落在三种推演路径之一）

**迭代触发**：若某类判断（如频繁错判冰点转折）连续出现3次，则生成“参数校准建议”，提示检查情绪指标阈值。

### 5.2 个股复盘 Skill 评估指标
构建基于策略的模拟交易数据库：
- 记录每一条“明日作战指令”的策略类型和触发条件
- 次日根据真实行情，模拟执行该指令（不考虑用户实际是否执行）
- 统计每个策略的：执行次数、胜率、平均盈亏比、期望值、最大回撤
- 细分统计不同市场阶段下该策略的表现（主升期胜率 vs. 退潮期胜率）

**迭代触发**：当某策略连续多笔期望值为负，或特定环境（如退潮期）中胜率骤降，系统会发出“策略失效预警”，并建议在该环境下暂停使用此类指令，或要求收紧触发条件。

### 5.3 用户执行反馈
允许用户简单打卡：“今日完全按计划执行 / 部分执行 / 完全未执行”。系统统计执行偏差率，长期高于阈值时，会生成**“执行纪律风险报告”**，提醒用户最大的问题在于人而非策略。

## 六、使用流程示例

**交易日 15:30**
> 用户：“开始今日复盘。”
> 
> 市场复盘 Skill：“今日复盘摘要：大盘缩量震荡，情绪回落至‘恐惧’，主线人工智能板块内部大分歧，有退潮征兆。明日作战地图：若低开破3050则进入防御，总仓位降至3成以下……”
> 
> 个股复盘 Skill：“现在开始个股复盘，我将引入今日市场环境：退潮预警、3成仓上限。”
> 
> 用户输入：“复盘 浪潮信息，成本32.5”
> 
> 个股 Skill：“【浪潮信息】技术面：今日在关键支撑31.8处缩量止跌……逻辑状态持平，但板块内已沦为跟风。纪律提示：您今日有卖出动作，与昨日‘持有等待’计划不符，属情绪化交易。明日预案：鉴于当前退潮预警，不具备新开仓条件。若明日反弹至33.2但无量，则清仓离场；若跌破31.8，立即止损。不开新仓。”
> 
> 系统后台自动记录这些分析，并将明日的实际走势纳入验证库。

---

这个设计最终交付的是一个“分析-执行-评估-进化”的完整闭环，使 AI 不再是静止的问答工具，而是你交易系统中一个不断成长、有据可查的智能组件。
